Project: Multiple Image Upload Component for GCP with Integrations of 5 Projects Through APIs (Laravel + PHP + Typescript + Vue.js + Tailwind + GCP + APIs) (2024/July)

A multi-image upload component was implemented in three integrated frontend applications, all sharing the same base stack of TypeScript and Vue.js. Images are sent to a backend engine in Laravel, which processes the images and uploads them to a Google Cloud Storage (GCS) bucket. The images are identified by their hash, and their storage URL are returned in the payload of a fifth system. Those URLs are then saved in the database using Eloquent for future reference.

Project: Automated Reviews Request (Laravel + PHP + Eloquent + Twilio) (2024/June)

Clients who have been assigned to a company may be requested to leave a review for that company. Currently, this process is done manually by running queries to identify eligible clients and then sending them SMS messages one by one. In this project, I developed an automated process to identify clients who were assigned in the last 72-96 hours and have not received a text message in the last 14 days. This is done using Eloquent through a Laravel job that runs once a day. Additionally, slightly different messages are created depending on the site the clients used, and these messages are sent at the appropriate times based on their time zones.

Project: Application to Optimize RFID Reader Processing (Java 8) (2015)

When working with RFID technology it is important to note that there is an RFID antenna responsible for capturing the transmit signal from the RFID tag chips and transmitting them to a reader that will process the signal. In this project, MOTOROLAS readers were used to process the labels and send the data to a middleware that was used as a mediator to transport the data to programs of different protocols using JavaEE. However, it was observed that excessive processing slowed down especially when the readings estimated at around 10 million tags read. Given this situation, an application was developed as a layer between middleware and enterprise software to optimize these readings.

Project: Data Reports from a Data Warehouse (JavaFX + Jasper Report) (2014)

The referenced data warehouse aims to store garment products in two types of cargo units: standard and beehive. The standard refers to boxes that store a single product while hives, on the other hand, allow for the storage of multiple product items. However, despite reporting the total number of boxes available, it does not specify what type of boxes are available for storage, nor whether all boxes are already occupied in the warehouse. In this context, an application using JavaFX was developed to solve this problem, allowing reports to be generated in order to inform the operator about the management, if there is a possibility of inclusion of a new product in the system and how many hives and standard free boxes are available.
